# AuraOne Open Community Channels

The shared community model keeps support predictable across the three flagships while preserving product-specific discussion spaces.

## Canonical Channels

| Channel                         | Purpose                                                         | Owner                | Safety notes                                                                                                   |
| ------------------------------- | --------------------------------------------------------------- | -------------------- | -------------------------------------------------------------------------------------------------------------- |
| GitHub Issues                   | Bugs, feature requests, docs issues, reproducible support cases | Product maintainers  | Do not post secrets, private prompts, traces, datasets, local paths, credentials, or security vulnerabilities. |
| GitHub Discussions              | Design discussion, RFC feedback, community Q&A                  | Product maintainers  | Keep support requests reproducible and move bugs to Issues.                                                    |
| Discord `#rubric-studio-open`   | Rubric Studio Open community chat                               | Developer Relations  | No private eval data or provider API keys.                                                                     |
| Discord `#robotics-studio-open` | Robotics Studio Open community chat                             | Developer Relations  | No proprietary datasets, lab videos, credentials, or export links.                                             |
| Discord `#agent-studio-open`    | Agent Studio Open community chat                                | Developer Relations  | No raw MCP traces, tool credentials, private prompts, or customer data.                                        |
| Discord `#open-studio-platform` | Shared platform, template, release, security-process questions  | Platform maintainers | Security reports must move to `security@auraone.ai`.                                                           |
| `security@auraone.ai`           | Private vulnerability disclosure                                | Platform Security    | Never request vulnerability details in public channels.                                                        |
| `partners@auraone.ai`           | Design-partner coordination for Cloud, Enterprise, and Programs | Developer Relations  | Do not route security reports or support tickets here.                                                         |

## Moderation And Contributor Safety

- Enforce the shared Code of Conduct.
- Remove secrets or private data from public channels when possible and tell the author to rotate affected credentials.
- Move vulnerability reports to the private disclosure process immediately.
- Keep commercial routing explicit: Cloud, Enterprise, Programs, and managed reviewer support are directed to the appropriate commercial intake, not handled in community chat.
